Daily Report

Project: SaaS Subscription Database


1. Work Completed

Today I designed and completed a relational database model for a SaaS subscription system.

I created tables for:
- Companies
- Users
- Plans
- Subscriptions
- Payments
- Support Tickets

I added relationships between tables using primary keys and foreign keys.


2. Database Relationships

The main relationships created were:

- One company can have many users.
- One company can have many subscriptions.
- One plan can be used by many subscriptions.
- One subscription can have many payments.
- One user can create many support tickets.


3. SQL Queries Created

I created relationship queries to test that the database connections work correctly.

Examples:
- Users with company information.
- Subscriptions with company and plan details.
- Payments with company, plan, and subscription status.
- Support tickets with user and company details.
- Companies without users.
- Subscriptions without payments.


4. Business Reports Created

I created manager-focused reports including:

- Total paid revenue.
- Revenue by company.
- Revenue by subscription plan.
- Active subscriptions by plan.
- Number of users by company.
- Support tickets by company.
- Open tickets by priority.
- Top companies by revenue.
- Average payment amount by plan.
- Executive summary report.


5. Database Validation

I tested the database using constraints.

The database prevents:
- Duplicate IDs.
- Invalid foreign key relationships.
- Missing required values.
- Invalid status values.


6. Skills Practiced

Today I practiced:

- Relational database design.
- Primary keys and foreign keys.
- INNER JOIN and LEFT JOIN.
- GROUP BY and aggregate functions.
- Creating business reports from database data.


7. Challenges

I improved my understanding of how database relationships affect query results.
